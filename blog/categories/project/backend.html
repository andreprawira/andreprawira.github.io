Backend - REST API - AWS BeanStalk

Clone this repository. It contains a REST API (developed with Spring Boot) with GET, POST, UPDATE, and DELETE methods including the configuration with AWS DynamoDB. Assuming you have an IAM with full administrator policies, you just have to generate access and secret keys and put those keys in lines 3 and 4. Afterwards, run the Spring Boot app and test all the methods with Postman to make sure it works. Open target folder and find AWS-DynamoDB-SpringBoot-0.0.1-SNAPSHOT.jar, you are going to upload it to BeanStalk.

Go to AWS console and search for beanstalk, click Get Started, fill the Application name as REST API and Choose Java as the platform. Choose upload your code for application code and upload the war file then click configure. Optionally, you can configure environment id (will be your url). If the application health status is "Ok" then go to configuration -> software -> scroll down to Environment properties then add PORT in the name and 9001 in the value. This way you will open port 9001 for testing with your BeanStalk app's url in postman.
