Deploying a Full Stack App on AWS

A little background to the blog, I was placed on a 6 months internal project when I first start working in YASH, this project is for one of the clients in the pharmaceutical business. There were several user stories and YASH divided them one team per user story, my team and I were working on a health study system. This blog will guide you how to deploy the diagram above on Amazon Web Services. I will provide you some of them codes and scripts but not all of them. That's why, this guide is more focused on deploying them apps, not developing.

This article will show you how to deploy a REST API developed in SpringBoot in AWS BeanStalk as the back-end with a React app in AWS S3 UI Bucket as the front end static page. We are going to use DynamoDB as the database, and use AWS Data Pipeline to funnel the information from DynamoDB to AWS S3 Back-up Bucket, pipeline will convert objects from DynamoDB into a JSON file. Furthermore, we will convert JSON files in the back-up bucket using AWS Glue to .parquet files. Doing so can reduce each file size by 70%, those parquet files are going to be stored in Converted Parquet Bucket. Finally, I will show you how to retrieve .parquet files using Cognito and a Python script. I will also show you how to store all the codes and documentations in AWS CodeCommit. However, before we get into all that, I will assume you already have set-up a working AWS account. For each service, I will create an article (dont worry they will all connect)
