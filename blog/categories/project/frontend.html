Frontend - React - AWS S3

As for frontend react app, I cannot share the project with you guys because I was not tasked developing frontend, I was only tasked to deploy the app that was already build by my co-workers. It was difficult for me to figure out how to deploy a front end app in AWS because there is no equivalent war file in React. Instead, React has to be build and you can use the built folder in S3 as a static page. You can also use BeanStalk to host your front end but I do not suggest doing that as it will cost you money. Hosting the React app in S3 will cost you $0 dollar, you can also configure error page for 404 get request. Anyway, lets get into it.

  • Create an S3 bucket and turn off Block all public access in the Permission tab
  • Open the bucket, navigate to Permission, click Bucket Policy and edit it
  • Add this policy in the editor

    { "Version": "2012-10-17", "Statement": [ { "Sid": "PublicReadGetObject", "Effect": "Allow", "Principal": "*", "Action": "s3:GetObject", "Resource": "arn:aws:s3:::NAME-OF-YOUR-BUCKET/*" } ] }

    doing this will make your bucket completely open for public
  • Since I did not provide you with my own React app, you can download a sample Calculator app
  • After you download it, open CMD and cd to the calculator folder.
  • Open package.json and scroll down until you see scripts and add

    "deploy": "aws s3 cp --recursive ./build s3://YOUR-BUCKET-NAME"

    between build and test
  • Execute npm run build in terminal, this will create a build folder in the project
  • Upload ONLY the build folder to the bucket by executing aws s3 cp --recursive ./build s3://YOUR-BUCKET-NAME
  • Go to properties -> enable static web hosting -> point out the index and error documents with index.html and error.html -> click the end point


    • In an actual project, your React app (frontend) will have to communicate with the REST API (backend), if your task is to develop only backend (like me), your front end team will develop the app and add the endpoints (URI for each requests GET, POST, PUT, DELETE, UPDATE) to the front-end code.
